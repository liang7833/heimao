# Kronos 交易系统打包说明

## 📦 快速开始

### 方法一：使用自动打包脚本（推荐）

```bash
python build_exe.py
```

### 方法二：手动使用 PyInstaller

```bash
pyinstaller --clean kronos_trading.spec
```

---

## 📋 打包内容

### 已包含的文件/目录

| 项目 | 说明 |
|-----|------|
| `gui_trading.py` | 主程序入口 |
| `models/` | 所有大模型（Qwen3.5、FinGPT、Kronos 等） |
| `Kronos/` | Kronos 核心模块 |
| `strategy_config.py` | 策略配置文件 |
| `train_config.yaml` | 训练配置文件 |
| 所有依赖库 | 自动打包进 exe |

---

## 🔧 训练模型导入导出

### 功能说明

打包后的 exe 仍然支持以下功能：

1. **模型导出**
   - 训练好的模型会保存到 `models/` 目录
   - 支持自定义导出路径

2. **模型导入**
   - 可以从外部导入训练好的模型
   - 支持多种模型格式

### 使用方式

打包后，模型文件的工作目录：
- **Windows**: `C:\Users\<用户名>\AppData\Local\Temp\_MEIxxxxxx\`
- 或者相对于 exe 的目录

## ⚙️ 配置说明

### 修改打包配置

编辑 `kronos_trading.spec` 文件：

```python
# 添加更多数据文件
datas = [
    ('你的目录', '目标目录'),
]

# 添加更多隐藏导入
hiddenimports = [
    '你的模块',
]
```

### 修改检查周期

编辑 `performance_monitor.py`:
```python
check_interval_seconds = 900  # 15分钟（默认）
```

## ⚠️ 注意事项

1. **首次运行**
   - 首次运行可能需要较长时间解压模型
   - 后续运行会快很多

2. **模型大小**
   - 打包后的 exe 会比较大（包含所有模型）
   - 预计大小：2GB-5GB

3. **杀毒软件**
   - 可能会误报，添加信任即可

4. **训练模型**
   - 训练好的模型建议备份
   - 重新打包不会丢失用户训练的模型

## 🐛 故障排除

### 问题：找不到模型文件

**解决方案**:
确保 models 目录在项目根目录，重新打包。

### 问题：exe 无法启动

**解决方案**:
1. 检查是否缺少依赖
2. 查看错误日志
3. 尝试以管理员身份运行

### 问题：训练模型无法保存

**解决方案**:
确保有写入权限，或修改保存路径到用户目录。
